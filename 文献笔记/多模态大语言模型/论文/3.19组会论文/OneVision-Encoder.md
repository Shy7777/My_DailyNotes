## 背景：
提出假说：通用人工智能本质上是压缩问题，当架构与数据结构对其时候扩展效果是最佳的，但是当前视觉信号高度冗余、而判别信息（意外性）是稀疏的，浪费了太多资源，所以解决视觉理解问题，必须要和架构与视频信息论原理对其，即提出codec。

方法：为了扩展视觉智能，架构必须与数据的信源结构相一致。在这项工作中，我们提出了 OneVision-Encoder（OV-Encoder），一种 **HEVC 风格**的视觉 Transformer，它将时空表示学习与视频信号的本征预测结构相一致，OV-Encoder 并非均匀处理密集像素网格，而是明确确定哪些视觉信号构成独立证据，并选择性地编码信号熵丰富的区域。为此，我们引入了 Codec Patchification，这是一种受编解码器启发的输入公式，它利用视频编解码器暴露的时序信号来组织视觉标记在块级别，并结合三维旋转位置嵌入（RoPE），联合编码空间和时序位置，以支持对不规则时空布局的连贯注意力。 具体而言：
	(a) 密集视频编解码器分块化：一种受编解码器启发的视频编码公式，利用 P 帧暴露的运动中心时间信号对密集视频输入中选定的视觉区域（3.1%-25%）进行分块，同时保留密集时间覆盖。
	(b) 块级分块化：一种受编解码器启发的时序分块化方案，将视频流分割为固定长度的块，并构建具有块级位置编码的块级表示。
	(c) 单图像空间分块化：编解码器分块化的空间实例化，为单图像输入构建块级表示，实现静态视觉内容的结构化建模。
OV-Encoder 提供了一种基于双向注意力机制的视觉编码器，能够有效地支持图像和视频理解。

## 创新点、前景、贡献：
	1.提出了 OneVision-Encoder (OV-Encoder)，一种 HEVC 风格的视觉 Transformer，通过Codec 块级编码，将时空表示学习与视频信号固有的预测结构相统一。
    2.我们引入了 Codec Patchification，一种受 Codec 启发的输入公式，利用 Codec 源生的时序信号，从密集视频中选取性地编码信息丰富的视觉块（3.1%-25%），同时通过 3D-RoPE 统一视频、块级采样和单图像输入。
	3.我们采用了一种自监督的聚类判别目标，通过大规模概念库联合建模对象级和运动级语义，实现结构化且模态无关的视觉表示学习。
	4.大量实验验证了 OV-Encoder 在各种评估设置中的有效性。在基于 LLM 的探测下，该模型在多模态基准测试中始终超越强大的视觉骨干网络，包括 Qwen3-ViT 和 SigLIP2。在注意力探测下，OV-Encoder 在 7 个基准测试中优于 SigLIP2 和 DINOv3，表明其具有强大的表征学习能力。

## 数据流：

### 路径 A：视频 codec 模式

1. 输入原始视频 ViVi​
    
2. 按 GOP 分解
    
3. 提取 I-frame、P-frame
    
4. 从 P-frame 得到 motion vector + residual
    
5. 对每个 patch 聚合运动强度与残差能量
    
6. 选 Top-r patch，得到 mask ΩΩ
    
7. I-frame 保留所有 patch，P-frame 只保留 ΩΩ 选中的 patch
    
8. 给每个 token 赋予 3D-RoPE 位置
    
9. 送入共享 ViT
    
10. 线性投影 + attentive pooling
    
11. 得到 video embedding
    
12. 与 video cluster centers 做 discrimination loss。

## 结果、结论：

在这项工作中，提出了 OV-Encoder，这是一个统一的自监督视觉框架，它摒弃了以帧为中心的建模方式，并将表征学习与视频信号的可预测结构相一致。通过将判别性视觉证据视为基于块级和以运动为中心的，OneVision-Encoder 能够在固定的 token 预算下选择性地编码信息丰富的区域，同时保持密集的时间覆盖。这一设计的核心是 Codec Patchification，它构建了稀疏但保留结构的时空 token 布局，并自然地扩展到基于注意力的编码器中的块级时间建模和单图像输入，该编码器由三维旋转位置编码支持。结合一个联合建模对象级和运动级语义的聚类判别目标，无需外部监督，OneVision-Encoder 在 LMM 探测和注意力探测下均实现了最先进的性能。这些结果突显了受编解码器启发的块级稀疏性作为通用视觉表征学习有效且可扩展基础的潜力。

## 补充知识：
### 什么是Codec Patchification技术：
**Codec Patchification 是一种编解码启发的稀疏分块技术：**

1. 从压缩视频提取**运动向量 + 残差**
2. 只选择**高信息 patch**（3%–25%）
3. 做**稀疏时空编码**，大幅降本增效
4. 让大模型像编解码器一样，**只看变化、不看静态**

### 什么是LLM probe：
**LMM Probe（Large Multimodal Model Probe，多模态大模型探针）** 是一种**轻量级、可解释的模型分析技术**，核心是用**小而简单的线性 / 浅层分类器**去 “探测”**冻结的多模态大模型（MLLM）** 内部的**特征表示（Representation）**，以诊断模型学到了什么、能力边界在哪、缺陷是什么。

**LMM Probe = 冻结大模型 + 训练轻量探针 + 用探针性能反推大模型的内部能力与缺陷。**

**LMM Probe 是多模态大模型的 “诊断工具”：**
1. **冻结**大模型，**不碰**主干参数
2. 插**轻量探针**（线性 / 浅层）
3. 训探针，用**准确率反推**模型能力
4. 低成本、高可解释、可复现，**打开黑箱**

### 什么是HEVC

**HEVC = High Efficiency Video Coding

高效率视频编码**

外号：**H.265**

一句话：

**目前最主流、最通用的视频压缩标准，比上一代 H.264 压缩率高 50% 左右。**

---

二、HEVC 到底在干什么？（超直观）

视频有**巨大冗余**：

- 空间冗余：一帧里很多像素长得一样
- 时间冗余：前后帧几乎没变化

HEVC 的目标：

**用最少的数据，存下一模一样的视频。**

---

HEVC 最核心的 4 个技术（必须记住）

1. 分块：CTU（Coding Tree Unit）

- 把画面切成**最大 64×64 的块**
- 内容简单就切大块，内容复杂切小块
    → 这就是你前面问的 **Patchification 的来源**
 2. 帧内预测 Intra Prediction

- 只用**当前帧**，通过周围像素预测中间像素
- 只存**预测误差**，不存原图
    → 消除**空间冗余**
 3. 帧间预测 Inter Prediction（最关键）
- 拿**前面的帧**当参考
- 找一块相似的区域，记录它**往哪移、移多少**
    → 这就是 **Motion Vector 运动向量 MV**
 3. 残差 + 熵编码
- 只存**预测和真实的差（残差 Residual）**
- 再压缩 → 体积暴减
---
四、HEVC 的三种帧（超级重要）
- **I 帧**：关键帧，自己独立编码，不参考别人
- **P 帧**：向前参考，只存运动向量 + 残差
- **B 帧**：前后都参考，压缩率最高
**P 帧 / B 帧体积只有 I 帧的几十分之一。**
---
五、和你正在看的论文技术有什么关系？
**Codec Patchification**
就是**直接用 HEVC 里的信息**：
- Motion Vector（运动大的地方重要）
- Residual（变化大的地方重要）
- CTU 分块（天然的 patch）

让大模型**只看视频里 “动了、变了” 的地方**，

**不看静态背景，大幅省算力。**

---

六、极简总结（可直接背）

**HEVC（H.265）是新一代高效视频压缩标准，

通过 CTU 分块、帧内 / 帧间预测、运动估计、残差编码，
大幅消除视频的空间与时间冗余，
比 H.264 体积小 50%，画质不变。
它输出的运动向量、残差、分块信息，
可直接用于多模态模型的高效编码（如 Codec Patchification）。**
