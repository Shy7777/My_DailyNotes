2.字母异位词分组问题
排序后的字符串作为哈希表的键，时间复杂度为O（n k log k）n个词，k是排序复杂度
```python
class Solution:
def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
	hashtable=defaultdict(list)
	for s in strs:
		s_str="".join(sorted(s))
		hashtable[s_str].append(s)
return list(hashtable.values())
```

# 知识点：

哈希表中，set和dict的区别和什么时候用：
**`set` 和 `dict`（哈希表）的选择取决于是否需要存储「键值对」**：

---

### 用 `set` 的场景：

- **只需要快速判断元素是否存在**（成员检测）
- **不需要关联额外信息**
- **典型操作**：`x in s`, `s.add(x)`

**示例**：  
`hashtable = set(nums)` 是为了 O(1) 时间判断某个数字是否存在，从而检查连续序列（如 `num+1`, `num-1` 是否在数组中）。

---

### 用 `dict` 的场景：

- **需要通过键（key）关联值（value）**
- **典型操作**：`d[key] = value`, `value = d.get(key)`

**经典例子**：

- 「两数之和」：用 `dict` 存储 `{数值: 下标}`
- 「前缀和」：用 `dict` 存储 `{前缀和: 出现次数}`

---

### 对比总结：

|需求|数据结构|原因|
|---|---|---|
|快速查存在性|`set`|更轻量，内存更少|
|查存在性 + 关联数据|`dict`|必须存储键值对|
